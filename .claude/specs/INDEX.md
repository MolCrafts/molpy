# Specs

- [incremental-typify-01-region](incremental-typify-01-region.md) — 增量分型 1/3（依赖 molrs region-support-01/02）。`AffectedRegion`（`Atomistic`/`CoarseGrain` 子类，IS-A MolGraph）：图编辑后由 `extract_subgraph(touched, radius)` 抽出的受影响区域（半径自适应=max(typifier.context_radius,4)），带 interior/boundary/entity_map + **结构 `__hash__`/`__eq__`**（molrs structural_hash/is_isomorphic）。生产者 Reacter/Crosslinker 从 molrs.Reaction.apply 返回的 touched handle 构建；取代扁平 ReactionResult.modified_atoms；因是 MolGraph 可直接喂 AmberTools。架构 notes/incremental-typification-design.md [done]
- [incremental-typify-02-cache](incremental-typify-02-cache.md) — 增量分型 2/3（依赖 01）。`typify_region`（把区域当独立 Atomistic 分型，boundary 壳层给全环境，读 interior 类型，按 molrs canonical_order 排）+ `RetypeCache`（按结构哈希去重，is_isomorphic 确认碰撞）。重接 `_incremental_typify`：以区域+缓存取代整图原子分型 → 消除高分子增长 O(N²)（降到 O(#distinct junction)）；region 缺失回退旧路径 [done]
- [incremental-typify-03-integration](incremental-typify-03-integration.md) — 增量分型 3/3（依赖 02）。把区域缓存接到三处：polymer builder（全程共享 RetypeCache → junction 跨链去重，O(N²) 修复落地）、crosslink（可选 typifier= 局部重分型，默认仍纯拓扑）、AmberTools（区域子图经 antechamber 分型、按哈希缓存、GAFF 类型经 entity_map 写回；Atomistic→PDB 桥不变）[done]

- [crosslink-01-deterministic](crosslink-01-deterministic.md) — 离线交联 chain 1/3（`builder/crosslink/`，**依赖 molrs 链 reaction-smarts-01/02**）。molpy 只做交联编排——匹配/SMIRKS 应用/图编辑/距离全在 molrs：`molrs.Reaction(smirks)` 解析+应用、`SmartsPattern.find_matches_mapped` 匹配、`NeighborList` 距离、`topo_distances` 主链排序、Atomistic.copy。`Crosslinker(ABC)`（house-style 仿 `VirtualSiteBuilder`：`__init__(reaction, *, cutoff)`、`apply(graph)→新 graph` copy-once+molrs 匹配+循环 hook `select`+molrs.apply+返回，无 classmethod、**无 molpy SMARTS 引擎/无 MatchSet/Block**）+ `DeterministicCrosslinker`（穷尽 100%/`spacing` 均匀交联点/显式 `pairs`，无 RNG，`exclude_*`）。cutoff 有坐标按距离、无坐标退拓扑，xyz 可选 [done]
- [crosslink-02-random](crosslink-02-random.md) — 离线交联 chain 2/3（依赖 01）。`RandomCrosslinker(Crosslinker)` **只重写 `select`**：`numpy.random.RandomState(seed)` 打乱基类候选、消耗到目标 `conversion`（=已反应/限制反应物位点，±1 步）、遵守 `cutoff`/`exclude_same_molecule`/`exclude_same_match`/`max_per_molecule`；Flory–Stockmayer 随机成网，seed 可复现，`conversion=1.0` 无候选提前停不死循环。匹配/距离/改图全继承基类（→ molrs），不重造 [done]
- [crosslink-03-builder](crosslink-03-builder.md) — 离线交联 chain 3/3（依赖 02）。①`PortMatcher` 读 `atom["port"]`（BigSMILES `<`/`>`/`$` 或 `[*:n]`）成与 molrs 匹配**同形**的 occurrence（`list[dict[map:handle]]`）——建模标好的位点与 SMARTS 现找的位点合流，同一 `Crosslinker` 消费（选点前端可换、molrs 编辑引擎唯一）。②端到端 **PEO 凝胶配方**（build→crosslink[spacing/conversion]→`molpy.optimize` minimize→export）。③builder 收敛分析：`PolymerBuilder`=match(port)/pair(相邻单体)/edit(Reacter.run)=退化交联；建议 bonding 走 molrs、placement/sequence 留 builder，**不重写能跑的 builder**（Connector 迁移列 follow-up，零回归） [done]
- [graph-assembler-01-reach](graph-assembler-01-reach.md) — 装配重构 1/4（**不依赖 molrs 新 API，可立即落地**）。修科学正确性：区域重分型的两个半径**不是两个自由度**——`retype_reach ≡ context_reach ≡ reach`（"编辑落进 a 的判定球"与"a 需要多大的球"是同一个量的两种看法），写回集合 = `ball(touched, interior_reach)`，抽取 = `interior_reach + reach`。现状单一 `radius=R` ⟹ 写回 `ball(R-1)` 而最外圈只剩 1 层上下文，**仅在 `reach=1,R=2` 时正确**（`ForceFieldTypifier` R=4、`AmberToolsTypifier` R=2 均错）。环谓词并进 `reach`（`⌊环系尺寸/2⌋+1`）而非只并进 context——否则成环编辑漏重分型。`TypeScope(reach)` 单字段 + `TERM_REACH=2`（二面角元数减二，非魔数）；`RegionTypes` 补 impropers；写回过滤器从 `not in boundary` 换成 per-atom hops；删 `_FLOOR`/`region_radius`/三处 `context_radius`。oracle = 全图分型（类型的*定义*） [draft]
- [graph-assembler-02-kernel](graph-assembler-02-kernel.md) — 装配重构 2/4（依赖 01，**不依赖 molrs 新 API**）。**PolymerBuilder 与 Crosslinker 合并**：二者只有"配对"不同（CGSmiles 邻接给显式边表 vs cutoff 内搜索采样；`DeterministicCrosslinker` 本就支持 `pairs=`）。合并证据链：①`place` 不是类差别是构造器参数 `placer=`（"两组分相对坐标有无意义"，交联悬空分子同样要摆）；②`_paste` 不是 hook 是**隐藏信道**——`select` 靠魔法字段 `monomer_node_id` 知道"哪些原子属于第 i 个节点"，事后 `cleanup_build_markers` 擦掉；正解是世界作**输入**，`MonomerLibrary.expand()` 打**正则 `RES_ID`/`RES_NAME`**（单体即残基，身份变成 PDB/prmtop 需要的输出而非待擦垃圾）。⟹ 具体类 `GraphAssembler` + 策略族 `Selector(ABC)`：`TopologySelector`/`ProximitySelector(ABC)`→`Exhaustive`/`Spacing`/`ExplicitPair`/`Random`（顺带让 `DeterministicCrosslinker` 三个互斥 kwarg 的非法状态不可表示）。**但实现合并 ≠ API 拆开**：`PolymerBuilder(MonomerLibrary, reaction, typifier=, placer=).build("{[#EO]|1000}")` **保留为 `GraphAssembler` 的真子类**（拥有单体库 + CGSmiles 语义，不是门面；对照被删的自由函数 `crosslink_gel`：不拥有数据、不做决定）；`Crosslinker`/`Deterministic`/`Random` 三个类消失（它们只持 selector 并把 `apply` 转发成 `assemble`）；交联 = `GraphAssembler(rxn, typifier=t).assemble(melt, RandomSelector(...))`。**selector 是 `assemble` 的实参不是构造器参数**——CGSmiles 拓扑只有 `build()` 时才知道，且构造器持 selector 会让每条拓扑作废缓存。**匹配在内核（O(N)一次），配对在选择器**（O(N²) 的是配对不是匹配）。**`RegionPatchCache` 必须是实例属性**——早期草案写在 `assemble()` 里，100 条链 100 个缓存，其自身 ac-005 必挂。O(N²) 有**四个**独立来源（`left.copy()`、`_build_entity_map`、`_preserve_node_ids`、`_remap_ports_registry`），根因是"拷贝+entity_map 重映射"表达图编辑。区域须在**全部 apply 之后**建；`RegionPatch(types+terms)` 两趟写回保证重叠幂等。电荷：不收 `charges=`——`ChargeModel.freeze()` 模板层折叠帽电荷归零 ⟹ 删的原子不带电，净电荷守恒成为**定理**，`_conserve_leaving_charge` 删。**铁律 6 re-export 审计**（`molpy.Reaction is molrs.Reaction`）。**修现存 bug** `_find_component` 找不到 map_number 时 `return 0`（畸形 SMIRKS 静默当成"位点在第 0 组分"→ 应 raise，ac-022）。前置：须先验证 molrs `apply(refresh=False)` 删原子时连带删关联键项 [draft]
