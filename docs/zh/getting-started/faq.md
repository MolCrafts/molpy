# 常见问题解答

MolPy 的定位：它解决了什么问题，和现有工具是什么关系，什么时候该用别的工具。

## 为什么需要 MolPy？

准备一个复杂的分子系统，少不了编辑拓扑、分配力场参数、生成引擎输入文件这些步骤。问题在于，这些步骤分散在不同的工具里，各有各的数据模型和暗含的约定，拼凑出来的跨包工作流脆弱不堪，绑死在特定引擎上，还很难复现。

MolPy 把整个准备层统一到一套显式表示里，设计围绕三个原则：

1. **统一数据模型。** `Atomistic`、`Frame`、`Block`、`Box` 自带显式的拓扑和类型化属性——从第一次解析到最终导出，全程用同一套对象干活，不管是聚合物、电解质、晶体还是混合物。
2. **力场就是数据。** 参数放在可查询的结构里，跟化学结构分开。这样就能在昂贵的模拟启动**之前**检查验证模型，不用等算崩了才回头找原因。
3. **显式编辑。** 构建、填充、聚合、基于反应的变换，全是写死的程序化操作，各步骤之间没有隐藏状态。

MolPy 不取代专门的工具包——它的角色是准备层，让这些工具彼此还能拼得起来。下面逐个对比，把分工说清楚。

## MolPy 对比其他工具

### ...ASE？

ASE 擅长表示原子结构和对接量子/经典引擎。它的 `Atoms` 对象装了坐标、元素种类、晶胞和计算器——但成键拓扑不是一等公民，和 MolPy 不同。

* **数据模型**: MolPy 同时管坐标、拓扑和周期性盒子；ASE 更偏原子列表加计算器对象。
* **力场**: ASE 给的是计算器，没有力场管理；MolPy 提供类型化器、成键项和参数容器。
* **编辑与构建**: MolPy 的聚合物构建、填充和反应依赖生成式拓扑；ASE 只把成键当作描述性信息。

两者互补。在 MolPy 里建好的系统可以导出到 ASE 去跑引擎。

### ...mBuild？

mBuild 是一个基于模板的框架，用可复用的构建块搭分子。

* **数据结构**: mBuild 用层级化的 `Compound` 对象；MolPy 是扁平的类型化结构，直接面向下游模拟。
* **力场**: mBuild 把类型化甩给 Foyer；MolPy 把类型化作为一等组件直接集成。
* **操作**: MolPy 提供显式的单体/端口/反应原语，语义直接面向模拟。

### ...ParmEd？

ParmEd 专门编辑**已有**的参数化拓扑——转换、修补、检查别的工具生成的系统。

* **方向**: ParmEd 加工现成的系统；MolPy 从零开始搭建。
* **数据模型**: MolPy 用统一的 `Frame`，没有残基、原子、参数各一套类的说法。
* **类型化**: MolPy 的类型化器框架跟具体力场解耦，基于规则驱动，不绑定到某个 MD 生态。

MolPy 生成的系统，到了需要 ParmEd 生态集成的环节，可以交给 ParmEd 做后处理。

### ...RDKit？

RDKit 是化学信息学的标杆开源工具包：SMILES 解析、构象生成、子结构搜索，样样在行。MolPy 通过[适配器层](../../api/adapter.md)和它互通，并在此基础上把视野拓宽到分子图之外：

* **面向模拟的结构**: 周期性盒子、成键 MD 拓扑、多相组装体。
* **系统级操作**: 聚合物构建、填充、反应——这些超出了 RDKit 的领域。
* **互操作性**: RDKit 分子和 MolPy 对象可以互相转换。

一句话：RDKit 管化学图，MolPy 管面向模拟的分子**系统**。

### ...MDAnalysis / MDTraj？

它们用来分析模拟**之后**的轨迹；MolPy 的重心在模拟**之前**——构建、编辑、类型化、填充。MolPy 的[计算](../compute/index.md)层也提供了一大票分析功能，底层是 Rust 内核，但数 TB 级别的后处理流水线还是 MDAnalysis / MDTraj 的天下。

## 什么时候该用别的工具？

MolPy 是做系统构建和参数化的工具包，不是万能药：

* **数 TB 级别的轨迹后处理**——去找 MDAnalysis 或 MDTraj。MolPy 的计算层只适合单系统分析，不是为大规模流水线设计的。
* **量子化学**——用 PySCF、ASE 加 QM 计算器，或者 Gaussian/ORCA。MolPy 不碰电子结构。
* **纯化学信息学**——直接用 RDKit。MolPy 通过适配器层把子结构搜索和构象生成这类任务交给它。
* **高通量筛选**——MolPy 以完全可控的方式构建单个系统；一次性筛几千个候选者，那是工作流引擎和 CADD 工具的事。
* **运行模拟任务**——MolPy 能生成输入文件，也能启动引擎，但执行管理本身是引擎的活，或者交给 Signac、AiiDA 这类工具管。
